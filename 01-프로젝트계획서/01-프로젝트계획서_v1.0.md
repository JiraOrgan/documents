# Jira 프로젝트 관리 시스템 프로젝트 계획서

## 1. 프로젝트 개요

### 1.1 배경 및 목적
- **배경**: 소프트웨어 개발팀의 애자일(Agile) 방법론 지원에 특화된 이슈 트래킹 및 프로젝트 관리 도구가 필요함. 팀 워크플로우 설계, 릴리즈 관리, 자동화, 보고까지 통합하는 협업 플랫폼 구축
- **목적**: Jira Software Cloud/Server 기반의 프로젝트 관리 시스템을 구축하여 이슈 관리, 스프린트 운영, 릴리즈 관리, 대시보드 등 통합 협업 환경 제공

### 1.2 프로젝트 범위

| 구분 | 내용 |
|------|------|
| 포함 범위 (In-Scope) | 이슈(Issue) CRUD, 워크플로우 엔진, 스크럼/칸반 보드, 백로그, 스프린트 관리, 대시보드, JQL 검색, REST API, 권한 관리, Audit Log, 릴리즈/버전 관리 |
| 제외 범위 (Out-of-Scope) | Confluence 연동, Marketplace 앱 개발, 온프레미스 Data Center 배포 |

# Jira 프로젝트 관리 시스템 화면 흐름 시퀀스 다이어그램

## 1. 전체 화면 흐름도

```mermaid
flowchart LR
    LOGIN[로그인] --> MAIN[메인 대시보드]
    MAIN --> BACKLOG[백로그]
    MAIN --> BOARD[보드<br/>스크럼/칸반]
    MAIN --> ROADMAP[로드맵]
    MAIN --> SETTINGS[프로젝트 설정]

    BACKLOG --> ISSUE_CREATE[이슈 생성]
    BACKLOG --> ISSUE_DETAIL[이슈 상세]
    BOARD --> ISSUE_DETAIL
    ISSUE_DETAIL --> COMMENT[댓글/첨부]
    ISSUE_DETAIL --> TRANSITION[상태 전환]

    MAIN --> SPRINT_MGMT[스프린트 관리]
    SPRINT_MGMT --> SPRINT_PLANNING[스프린트 플래닝]
    SPRINT_MGMT --> SPRINT_REVIEW[스프린트 리뷰]

    MAIN --> RELEASE[릴리즈 관리]
    MAIN --> REPORTS[리포트<br/>번다운/속도/CFD]
    MAIN --> SEARCH[JQL 검색]
    MAIN --> AUDIT[Audit Log]

    SETTINGS --> WORKFLOW[워크플로우 설정]
    SETTINGS --> PERMISSION[권한 관리]
    SETTINGS --> AUTOMATION_SET[자동화 규칙]
    SETTINGS --> LOGOUT[로그아웃]
    LOGOUT --> LOGIN
```

## 2. 주요 시퀀스 다이어그램

### 2.1 로그인 흐름

```mermaid
sequenceDiagram
    actor U as 사용자
    participant V as 화면
    participant A as API Server
    participant D as Database

    U->>V: 로그인 페이지 접근
    V-->>U: 로그인 폼 표시
    U->>V: 이메일/비밀번호 입력
    V->>A: POST /auth/login
    A->>D: 사용자 인증 확인
    alt 인증 성공
        D-->>A: 사용자 정보 + 역할(RBAC)
        A-->>V: JWT Token (Access + Refresh)
        V-->>U: 메인 대시보드로 이동
    else 인증 실패 (5회 초과)
        D-->>A: 실패
        A-->>V: 계정 잠금 에러
        V-->>U: "계정이 잠겼습니다 (30분)" 메시지
    else 인증 실패
        D-->>A: 실패
        A-->>V: 에러 응답
        V-->>U: 에러 메시지 표시
    end
```

### 2.2 이슈 생성 및 스프린트 배정 흐름

```mermaid
sequenceDiagram
    actor PO as Product Owner
    participant BL as 백로그 화면
    participant A as API Server
    participant D as Database

    PO->>BL: 이슈 생성 버튼 클릭
    BL-->>PO: 이슈 생성 폼 표시
    PO->>BL: 이슈 정보 입력<br/>(타입, 제목, 설명, 우선순위, SP)
    BL->>A: POST /rest/api/3/issue
    A->>D: 이슈 저장 (상태: Backlog)
    D-->>A: 이슈 키 (PROJ-143)
    A-->>BL: 생성 완료
    BL-->>PO: 백로그에 이슈 추가됨

    Note over PO,D: Sprint Planning 시점
    PO->>BL: 이슈를 스프린트로 드래그
    BL->>A: PUT /rest/api/3/issue/PROJ-143 (sprint_id 설정)
    A->>D: 스프린트 배정 + Audit Log 기록
    A-->>BL: 상태 → Selected for Sprint
```

### 2.3 워크플로우 전환 흐름 (개발 → 코드리뷰 → QA → Done)

```mermaid
sequenceDiagram
    actor DEV as Developer
    actor REV as Reviewer
    actor QA as QA Engineer
    participant BD as 보드 화면
    participant A as API Server
    participant WF as 워크플로우 엔진
    participant N as 알림 서비스

    DEV->>BD: 이슈 카드를 "In Progress"로 이동
    BD->>A: POST /transitions (→ In Progress)
    A->>WF: 전환 규칙 검증
    WF-->>A: 허용
    A-->>BD: 상태 변경 완료

    DEV->>BD: PR 생성 후 "Code Review"로 이동
    BD->>A: POST /transitions (→ Code Review)
    A->>WF: 조건 확인 (PR 링크 존재?)
    WF-->>A: 허용
    A->>N: Slack #code-review 알림 + 리뷰어 자동 지정
    A-->>BD: 상태 변경 완료

    REV->>BD: 코드 리뷰 승인
    BD->>A: POST /transitions (→ QA)
    A-->>BD: 상태 변경 완료

    QA->>BD: 테스트 실행
    alt 테스트 통과 (DoD 충족)
        QA->>BD: "Done"으로 이동
        BD->>A: POST /transitions (→ Done)
        A->>WF: DoD 체크리스트 검증
        WF-->>A: 전체 충족 확인
        A-->>BD: 상태 변경 완료
    else 테스트 실패
        QA->>BD: "In Progress"로 반려
        BD->>A: POST /transitions (→ In Progress)
        A->>N: 담당 개발자에게 알림
        A-->>BD: 상태 변경 완료
    end
```

### 2.4 릴리즈 관리 흐름

```mermaid
sequenceDiagram
    actor PM as PM
    participant RL as 릴리즈 화면
    participant A as API Server
    participant D as Database

    PM->>RL: 새 버전 생성 (v1.0.0)
    RL->>A: POST /rest/api/3/version
    A->>D: 버전 저장 (Unreleased)
    A-->>RL: 버전 생성 완료

    PM->>RL: 이슈에 Fix Version 지정
    RL->>A: PUT /rest/api/3/issue/{key} (fixVersions)
    A-->>RL: 업데이트 완료

    Note over PM,D: 배포 시점
    PM->>RL: 버전 릴리즈 처리
    RL->>A: PUT /rest/api/3/version (Released)
    A->>D: 상태 변경 + 릴리즈 노트 자동 생성
    A-->>RL: 릴리즈 완료 (포함 이슈 목록 표시)
```

**인터랙션**:
- 이메일 입력 후 포커스 이탈 시 실시간 중복 체크 (GET /api/v1/auth/check-email)

### 7.2 AI 요약 패널 (확장)

```
  📊 이번 레슨 AI 요약
  ┌────────────────────────────────────────┐
  │ 핵심 개념 3가지:                        │
  │ 1. N+1 문제: 연관 엔티티 N번 추가 조회 │
  │ 2. 해결책: FETCH JOIN or @EntityGraph  │
  │ 3. 배치 사이즈: hibernate.batch_size   │
  │                                        │
  │ [전체 요약] [마인드맵] [플래시카드]     │
  └────────────────────────────────────────┘
```

### 7.3 집중 모드 (GNB + 커리큘럼 숨김)

```
+----------------------------------------------------------+
|  [← 나가기]  JPA N+1 문제 해결         [집중 모드 OFF]  |
+----------------------------------------------------------+
| +----------------------------------+ +----------------+  |
| |  ▶  강의 영상 플레이어 (전체폭) | | 🤖 AI 튜터     |  |
| |  ...                             | | ...            |  |
| +----------------------------------+ +----------------+  |
```

**인터랙션**:
- 추천 질문 칩 클릭 → 자동으로 inp-chat에 입력 후 전송
- AI 응답 스트리밍 중 입력창 비활성화 (SSE 완료 후 재활성화)
- "더 쉽게" → 현재 대화에 level=1 파라미터 추가 재요청
- "더 깊이" → level=3 파라미터 추가 재요청
- 레벨 배지 (Level 1/2/3) 우상단 표시
- 피드백 👍/👎 클릭 → 저장 완료 토스트

### 13.3 FinOps Hard Limit 경고 배너

```
+----------------------------------------------------------+

|  Today: $152.40 / $150 (Hard Limit)                     |
|  현재: Haiku 전용 + 배치 작업 중단 중                    |
|  [수동 해제]  [FinOps 상세 보기]                         |
+----------------------------------------------------------+
```

**인터랙션**:
- 30초마다 자동 갱신 (GET /api/v1/admin/ai-ops/summary)
- Kill-switch 수동 조작 클릭 → 확인 모달 표시
- FinOps 패널 "FinOps 상세" → SCR-305
- AI 품질 "상세 보기" → SCR-304
- Observability "Observability" → SCR-306
- PII Output 탐지 건 클릭 → SCR-307 감사 로그 (해당 이벤트 필터)
- Hard Limit 초과 시 최상단 경고 배너 표시 (dismissible 불가)

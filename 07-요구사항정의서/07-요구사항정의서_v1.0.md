### 1.2 요구사항 상세

#### FR-001: 이슈 CRUD

| 항목 | 내용 |
|------|------|
| ID | FR-001 |
| 분류 | 이슈 관리 |
| 우선순위 | 필수 |
| 설명 | 사용자는 이슈(Epic/Story/Task/Bug/Sub-task)를 생성, 조회, 수정, 삭제할 수 있다 |

**상세 조건**:
- [ ] 이슈 제목은 `[모듈] 기능 요약` 형식 권장
- [ ] Epic은 2주 이상 소요되는 큰 기능 단위
- [ ] Story는 1 Sprint 내 완료 가능한 사용자 기능 (SP ≤ 8 권장)
- [ ] Bug 등록 시 재현 절차, 기대 결과, 실제 결과, 환경 정보 필수
- [ ] Sub-task는 상위 Story/Task의 세부 병렬 작업

**관련 화면**: SCR-006 (이슈 상세), SCR-007 (이슈 생성)
**관련 API**: POST/GET/PUT/DELETE /rest/api/3/issue

---

#### FR-013: 표준 워크플로우 6단계

| 항목 | 내용 |
|------|------|
| ID | FR-013 |
| 분류 | 워크플로우 |
| 우선순위 | 필수 |
| 설명 | 표준 워크플로우 6단계 (Backlog → Selected for Sprint → In Progress → Code Review → QA → Done) |

**상세 조건**:
- [ ] In Progress → Code Review: PR 생성 완료 시
- [ ] Code Review → In Progress: 리뷰어 변경 요청 시
- [ ] Code Review → QA: 리뷰어 승인 완료 시
- [ ] QA → In Progress: 테스트 실패 시
- [ ] QA → Done: DoD 체크리스트 전체 충족 시

**관련 화면**: SCR-004 (스크럼 보드)
**관련 API**: POST /rest/api/3/issue/{issueKey}/transitions
